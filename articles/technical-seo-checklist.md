# テクニカルSEOでまず押さえたいポイント  
（Stanislav.jp スタイル：短く・整理されていて・実務的）

こんにちは、Stanislavです。  
サイトの相談をいただくと、最初に必ず確認するのが **テクニカルSEO（技術面の土台）** です。

理由はとてもシンプルで、

> **土台が整っていないサイトは、どれだけ良いコンテンツを載せても評価されにくいから。**

このページでは、普段のコンサルやミーティングでも一番よく使う  
「まずここだけ押さえれば十分」というポイントだけに絞って整理しました。

---

# ✔ 1. Google がサイトを正しく読めているか？

## ▸ robots.txt の確認
まず、Google がそもそもサイトを読める状態かチェックします。

- 管理画面（/admin/）などはブロックできている？  
- 公開すべきページを誤って止めていない？  
- テスト環境（/staging/ 等）は検索に出ていない？

**ありがちなミス：**  
本番サイトの一部を `Disallow:` で止めてしまい、順位が上がらない。

---

## ▸ noindex が必要のない場所についていないか
本来検索に出すべきページに `noindex` が残っているケースは非常に多いです。

```
<meta name="robots" content="noindex">
```

誤設定が多いページ：
- ブログカテゴリ
- LP
- 商品ページ

「気づかないうちに noindex だった」問題は本当に多いので、月1回の点検がおすすめです。

---

## ▸ sitemap.xml が最新か？
- 新規ページ追加時に自動更新されている？  
- Search Console に送信済み？  
- 削除済みページ（404）は含まれていない？

sitemap が古いと Google が重要ページを見つけられず、順位が不安定になります。

---

# ✔ 2. URL と内部リンクがわかりやすいか？

## ▸ URL構造はシンプル？
NG: `/blog/post-123`  
OK: `/blog/technical-seo-checklist`

短くても意味が伝わる URL が最強です。

---

## ▸ 内部リンクで迷わない導線が作れている？
- 重要ページに自然にアクセスできる  
- 孤立（オーファン）ページがない  
- パンくずリストがある  

サイト全体の評価にも影響する部分です。

---

## ▸ canonical が正しく設定されている？
- パラメータ付きURLが増殖していない？  
- 重複ページは正規URLへ統一できている？

canonical は「どれが本物のページか」を Google に伝える仕組みです。

---

# ✔ 3. ページ速度とUX（ユーザー体験）

日本のユーザーは本当にスピードに敏感。  
1秒遅いだけで離脱が一気に増えます。

## ▸ Core Web Vitals の目安
- **LCP**：2.5秒以内  
- **CLS**：0.1 未満（レイアウトのズレ）  
- **INP**：200ms 以下（クリック反応速度）

---

## ▸ 速度改善の定番チェック
- 画像 → WebP / 適切なサイズ  
- 不要なJS・CSSを削減  
- `loading="lazy"` で画像の遅延読み込み  
- ファーストビューを軽量化  
- CDN（Cloudflare等）を使用

---

## ▸ モバイル表示の最終チェック
- タップ領域が小さすぎない？  
- フォントが小さくない？  
- 読み込み中に要素がズレない？  

UX は SEO に直結します。

---

# ✔ 4. HTTPS とセキュリティ

SEOというより「信頼性」の基本ですが、非常に重要です。

## ▸ HTTPSが全ページで統一されている？
- HTTP → HTTPS に 301リダイレクト  
- 混在コンテンツ（HTTP画像）がない  
- 古いスクリプトを使っていない

Chromeの警告はユーザー離脱に直結します。

---

# ✔ 5. 構造化データ（Schema.org）

ページ内容を Google に「意味」として伝える仕組みです。

## ▸ よく使うスキーマ
- Article  
- FAQPage  
- BreadcrumbList  
- LocalBusiness  
- HowTo  

## ▸ チェックする点
- JSON-LD 形式で統一  
- Rich Results Test でエラーなし  
- 内容とマークアップが正しい

構造化データは CTR を上げる効果もあります。

---

# ✔ 6. 定期メンテナンス（四半期に1回が理想）

テクニカルSEOは“一度やって終わり”ではありません。  
サイトもGoogleも常に変わるため、定期点検がおすすめです。

## ▸ 使用ツール
- Search Console  
- PageSpeed Insights  
- Lighthouse  
- アクセス解析（離脱ポイント確認）

---

# ◆ まとめ：テクニカルSEOは「目立たないけど、最も効く土台」

- Google が読みやすい  
- ユーザーが使いやすい  
- 更新しても崩れにくい  
- 長期的に順位が安定する  

テクニカルSEOは、サイト運営の「基礎工事」です。  
ここが整っているサイトは、必ず強くなります。
